# Edison My Sun: Mock App

A mock solar energy dashboard with an AI assistant, built for the GIPES multidisciplinary project. It simulates the Edison My Sun app experience: a homeowner can view their solar production, battery, grid, and consumption data and interact with an AI energy coach via chat.

## What It Does

- Displays a real-time solar energy dashboard (production, battery, grid, consumption, self-consumption %, temperature, active appliances)
- An AI assistant (Edison AI) answers questions about energy data in natural language
- The AI can update dashboard values, generate new chart layouts, and preview changes before they are applied
- All energy data is **mocked**, no real API or database is used

## Architecture

```
Browser
  └── Next.js Frontend (React 19)
        ├── InverterDashboard   — renders agent state
        ├── DashboardPreview    — accept/decline AI-suggested changes
        └── localStorage        — persists dashboard overrides

Next.js API Route (/api/copilotkit)
  └── CopilotRuntime + LangGraphAgent
        └── proxies to Python agent at localhost:8123

Python LangGraph Agent (agent/)
  ├── OpenAI gpt-4o-mini         — the LLM
  ├── CopilotKitMiddleware        — syncs state to/from frontend
  ├── get_dashboard tool          — reads state or returns mock data
  ├── update_dashboard tool       — writes new values into agent state
  └── generate_a2ui tool          — tells frontend to render a new chart layout
```

### Where Mock Data Comes From

All mock data is defined in `agent/src/dashboard_tools.py`, nothing comes from a real API:

| Data                              | Source                                                   |
| --------------------------------- | -------------------------------------------------------- |
| Solar, battery, grid, consumption | Hardcoded defaults in `src/app/page.tsx`                 |
| Weekly consumption & production   | Hardcoded fallback lists in `get_dashboard()`            |
| Appliance stats                   | Hardcoded fallback list in `get_dashboard()`             |
| Hourly energy prices (7 days)     | Generated by `_mock_energy_prices()` rule-based          |
| Hourly appliance usage (7 days)   | Generated by `_mock_appliance_hourly_usage()` rule-based |

## Prerequisites

- Node.js 18+
- Python 3.11+
- [uv](https://docs.astral.sh/uv/) (Python package manager)
- OpenAI API key

## Getting Started

1. Install dependencies:

```bash
npm install
```

This also installs the Python agent dependencies via `uv sync`.

2. Set up environment variables:

```bash
cp .env.example .env
```

Edit `.env` and add your OpenAI API key:

```
OPENAI_API_KEY=your-openai-api-key-here
```

3. Start the development server:

```bash
npm run dev
```

This starts both the Next.js frontend (port 5173) and the Python agent (port 8123) concurrently.

## Tech Stack

- **Frontend**: Next.js 16, React 19, Tailwind CSS 4, Recharts, shadcn/ui
- **Agent**: LangGraph (Python), LangChain, OpenAI gpt-4o-mini
- **AI integration**: CopilotKit v2 (React hooks + Python middleware)
- **Charts / generative UI**: A2UI the agent generates chart layouts declaratively at runtime

## A2UI - Agent-Driven Chart Generation

The app uses [A2UI]([https://a2ui.org/specification/](https://a2ui.org/#specification-versions)) for dynamic chart rendering. When the user asks for a chart, the agent calls `generate_a2ui`, which returns a declarative component tree. The frontend renders it using the catalog defined in `src/app/declarative-generative-ui/`.

**Key files:**

| Purpose                             | Path                                               |
| ----------------------------------- | -------------------------------------------------- |
| Component definitions (Zod schemas) | `src/app/declarative-generative-ui/definitions.ts` |
| Component renderers (React)         | `src/app/declarative-generative-ui/renderers.tsx`  |
| Catalog registration                | `src/app/layout.tsx`                               |
| Agent tool                          | `agent/src/a2ui_dynamic_schema.py`                 |

## Troubleshooting

**"I'm having trouble connecting to my tools"**

- Make sure the agent is running: `npm run dev:agent`
- Check that port 8123 is free
- Verify your `OPENAI_API_KEY` is set in `.env`

**Python import errors**

```bash
npm run install:agent
```
